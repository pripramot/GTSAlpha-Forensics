# ฝังโมเดลพิเศษ — แนวทางสำหรับผู้ใช้ (พื้นฐานผู้ใช้)

**เมื่อไหร่ใช้:** เมื่อตรวจแล้วพบข้อจำกัด (ข้อด้อย) ที่ต้องใช้ **โมเดลพิเศษ** (ฝึก/ปรับเอง) จึงฝังหรือใช้ในระบบ

เอกสารนี้เขียนให้ **ผู้ใช้** ทำตามเองได้ โดยอ้างอิงเครื่องมือที่มีอยู่แล้ว (เช่น Google Vertex AI) — ไม่ต้องพึ่งทีมพัฒนาแก้หลายที่

---

## 1. ทางเลือกหลัก: Google Vertex AI

ใช้เมื่อต้องการ **ฝึกโมเดลย่อย (sub-model)** หรือปรับโมเดล แล้วนำไปใช้/ฝังในระบบ

### สิ่งที่ต้องมี (พื้นฐานผู้ใช้)

- บัญชี Google Cloud และเปิดใช้ Vertex AI
- ติดตั้ง **Google Cloud SDK (gcloud)** บนเครื่อง
- ข้อมูลสำหรับฝึก (dataset) พร้อมแล้ว

### ตัวอย่างคำสั่ง (พื้นฐานผู้ใช้)

```bash
# ตั้งค่าโปรเจกต์ (ทำครั้งเดียว)
gcloud config set project YOUR_PROJECT_ID

# ทดสอบรันฝึกบนเครื่องก่อน (local-run)
gcloud ai custom-jobs local-run \
  --executor-image-uri=us-docker.pkg.dev/vertex-ai/training/pytorch-gpu.1-13.py310 \
  --local-package-path=./training_code \
  --script=train.py
```

แทน `YOUR_PROJECT_ID`, `./training_code`, `train.py` ตามโปรเจกต์และสคริปต์ฝึกของท่าน — รายละเอียดเต็มในลิงก์คู่มือด้านล่าง

### ขั้นตอนย่อ (ให้ผู้ใช้ทำเอง)

1. **ติดตั้ง gcloud CLI**  
   - ดาวน์โหลด: https://cloud.google.com/sdk/docs/install  
   - ติดตั้งแล้วรัน `gcloud init` และล็อกอิน

2. **ฝึกด้วย Custom Job (แนวทางอย่างเป็นทางการ)**  
   - อ่านคู่มือ: https://cloud.google.com/vertex-ai/docs/training/create-custom-job  
   - ใช้ Python script สำหรับฝึก แล้วส่งขึ้น Vertex AI ผ่าน gcloud หรือ Python SDK

3. **ทดสอบบนเครื่องก่อน (ถ้าต้องการ)**  
   - ใช้คำสั่ง `gcloud ai custom-jobs local-run` เพื่อรันฝึกบนเครื่องก่อนส่งขึ้น Cloud  
   - อ่าน: https://cloud.google.com/vertex-ai/docs/training/containerize-run-code-local

4. **Deploy โมเดลที่ฝึกแล้ว**  
   - อ่าน: https://cloud.google.com/vertex-ai/docs/predictions/deploy-model-api  
   - นำโมเดลที่ได้ไปฝัง/เรียกใช้ในระบบตามที่ออกแบบไว้

## 2. หลักการเขียนให้ผู้ใช้ทำเองได้

- **ไม่มั่ว:** อ้างอิงเฉพาะเอกสาร/คำสั่งที่มีอยู่จริง (ลิงก์ด้านบน)
- **พื้นฐานผู้ใช้:** สมมติว่าเป็นผู้ใช้ทั่วไป — ติดตั้ง gcloud ได้ มี dataset พร้อม
- **ลดการแก้หลายที่:** ให้ทำตามคู่มือ Google เอง; โปรเจกต์เราไม่ต้องคอยแก้สคริปต์ฝึกในหลายเครื่อง

---

## 3. ถ้าไม่ใช้ Vertex AI

- **เครื่องมืออื่น:** ใช้ framework อื่น (เช่น TensorFlow, PyTorch) ฝึกบนเครื่องหรือเซิร์ฟเวอร์ แล้ว export โมเดลไปฝังในระบบตามที่ออกแบบ
- **โมเดลสำเร็จรูป:** ถ้ามีโมเดลที่ผู้จำหน่าย/ชุมชนให้มาแล้ว ให้อ้างอิงคู่มือของโมเดลนั้น แล้วนำมา integrate ตาม API ที่ระบบรองรับ
